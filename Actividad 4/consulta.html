CONSULTA BASE DE DATOS

LAS VIEWS

Las vistas en MySQL son consultas SQL predefinidas almacenadas en la base de datos como objetos virtuales. Permiten a los usuarios acceder a conjuntos específicos de datos de manera conveniente y consistente, sin la necesidad de acceder directamente a las tablas subyacentes.

Procedimientos Almacenados:

Los procedimientos almacenados en MySQL son secuencias de comandos SQL que se almacenan en el servidor de la base de datos y pueden ser invocados y ejecutados mediante una llamada desde una aplicación o una consulta SQL. Son como un tipo de función

Diferencias entre NoSQL y MySQL

    Modelo de Datos:

  • MySQL utiliza un modelo relacional basado en tablas.
  • NoSQL utiliza diversos modelos de datos como documentos, clave-valor, columnares o de grafos.

    • Esquema:

  • MySQL tiene un esquema fijo y predefinido.
  • NoSQL puede tener un esquema dinámico o flexible.

    • Escalabilidad:

  • MySQL puede ser más difícil de escalar horizontalmente.
  • NoSQL es más fácil de escalar horizontalmente para manejar grandes volúmenes de datos.

    • Consultas y Transacciones:

  • MySQL utiliza SQL para consultas y admite transacciones ACID.
  • NoSQL utiliza interfaces específicas y puede no admitir transacciones ACID.

    • Flexibilidad y Agilidad:

  • MySQL puede ser rígido en cuanto a cambios en el esquema.
  • NoSQL ofrece más flexibilidad y agilidad para cambios en la estructura de los datos.

¿Que es MongoDB?

MongoDB es una base de datos NoSQL que utiliza un modelo de datos basado en documentos JSON. Es altamente escalable y flexible, permitiendo almacenar y consultar datos de forma eficiente en estructuras de documentos jerárquicas. MongoDB se destaca por su capacidad para manejar grandes volúmenes de datos, su facilidad de escalabilidad horizontal y su esquema flexible que permite adaptarse fácilmente a cambios en los requisitos de las aplicaciones.
